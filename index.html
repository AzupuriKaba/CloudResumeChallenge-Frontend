Project 1: The Cloud Resume Challenge

Description: Developed and deployed a static website to AWS s3 and cloudFront through CI/CD pipeline via API, Lambda in IaC. The challenge consists of two parts, a frontend and a backend. In the frontend, I developed a website, which hosted my resume using HTML and CSS. This is website is continuously deployed to AWS s3 bucket and CloudFront CDN through CI/CD pipeline that runs on GitHub Actions. For provisioning the backed infrastructure, a separate CI/CD pipeline that runs on GitHub action is used to automate the continuous deployment of AWS SAM / CloudFormation artifacts. A RESTful API is created in AWS API Gateway is powered by AWS Lambda (running Python) handles updates to a DynamoDB table. I integrated Py unit test framework to test the python code. Details of my solution to the Cloud Resume Challenge can be found here. Details of my experience participating in the challenge can be found at my blog.

Project 2: Deployed a Mult-tier website using AWS EC2
Industry: Internet related

Description: In this AWS project, I launched an EC2 instance without the need to invest in hardware up front, so that I can develop and deploy applications faster. I used Amazon EC2 to launch as many or as few virtual servers as needed, configured security and networking, and managed storage. Amazon EC2 enables us to scale up or down to handle changes in requirements or spikes in popularity, reducing our need to forecast traffic.

Project 3: Website Orchestration

Description: In this AWS project, I deployed a high-availability PHP application with an external Amazon RDS database to Elastic Beanstalk. Running a DB instance external to Elastic Beanstalk decouples the database from the lifecycle of our environment. This lets us connect to the same database from multiple environments, swap one database for another, or perform a blue/green deployment without affecting the database.

Project 4: Published Amazon SNS messages privately

Description: In this project, I worked on a hospital project to send reports online and developed a platform so the patients can access the reports via mobile and push notifications. I published the report to an Amazon SNS keeping it secure and private. The message was hosted on an EC2 instance within the Amazon VPC. By publishing the messages privately, we can improve the message delivery and receipt through Amazon SNS.
